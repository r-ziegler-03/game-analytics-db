## Session summaries

Past session summaries live in `session-summaries/`, named `YYYY-MM-DD-HHMM.md`. They're auto-loaded into context at the start of every session via a SessionStart hook — no need to read the directory manually.

When Ross indicates he's wrapping up (e.g. "wrap up", "end session", "that's it for today", "let's stop here"), or the conversation naturally seems to be concluding, write a new file in `session-summaries/` covering:
- What was worked on / decided this session
- Current state of the project (e.g. which milestone, what's done vs. pending)
- Any open questions or next steps for next time

Keep it tight — a few short paragraphs or bullets, not a transcript. If it seems like a session is winding down but Ross hasn't said so explicitly, offer to write the summary rather than assuming.
